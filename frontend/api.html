API Documentation

This page details how to use the xG prediction API.

Base URL

All API endpoints are relative to the following base URL:

https://redshaw-web-apps.onrender.com

Prediction Endpoint

This endpoint is used to get an xG prediction for a single shot. It requires a POST request with a JSON body containing the shot's features.

Endpoint

POST /redshaw-xg/api/predict

Request Body

{
    "x": 89.25,
    "y": 34.0,
    "situation": "OpenPlay",
    "shot_type": "RightFoot",
    "normalisation": {
        "is_normalised": false,
        "max_pitch_width": 68,
        "max_pitch_length": 105
    }
}

Parameters

  • x (float, required): The x-coordinate of the shot, normalised between 0 and 1.
  • y (float, required): The y-coordinate of the shot, normalised between 0 and 1.
  • situation (string, required): The game situation. Valid values are: OpenPlay, SetPiece, DirectFreekick, FromCorner, Penalty.
  • shot_type (string, required): The type of shot. Valid values are: Head, RightFoot, LeftFoot, OtherBodyPart.
  • normalisation (object, optional): An object for handling coordinate normalisation.
    • is_normalised (boolean): Set to false if providing raw coordinates that need normalisation, or true if coordinates are already normalised.
    • max_pitch_width (float): The pitch width to use for normalisation if is_normalised is false (does not need to be provided if is_normalised is true).
    • max_pitch_length (float): The pitch length to use for normalisation if is_normalised is false (does not need to be provided if is_normalised is true).

    Example: {"is_normalised": false, "max_pitch_width": 68, "max_pitch_length": 105}

Successful Response (200 OK)

{
    "xG": 0.08,
    "inputs": {
        "x": 0.85,
        "y": 0.5,
        "situation": "OpenPlay",
        "shot_type": "RightFoot",
        "normalisation": {
            "is_normalised": false,
            "max_pitch_width": 68,
            "max_pitch_length": 105
        }
    },
    "chosen_model": "advanced_model",
    "chosen_model_features": [
        "x",
        "y",
        "distance_to_goal",
        "angle_to_goal",
        "situation_OpenPlay",
        "shot_type_RightFoot"
    ]
}

Heatmap Grid Endpoint

This endpoint provides a pre-calculated grid of xG values for generating a heatmap. It's a GET request and takes query parameters to filter the data.

Endpoint

GET /redshaw-xg/api/predict/grid

Query Parameters

  • situation (string, optional): The game situation. Valid values are: OpenPlay, SetPiece, DirectFreekick, FromCorner. 'Penalty' is not supported for grid prediction.
  • shot_type (string, optional): The type of shot. Valid values are: Head, RightFoot, LeftFoot, OtherBodyPart.
  • max_length (float, optional): The maximum length of the pitch for scaling coordinates. Requires max_width.
  • max_width (float, optional): The maximum width of the pitch for scaling coordinates. Requires max_length.

Example Usage

/redshaw-xg/api/predict/grid?situation=OpenPlay&shot_type=RightFoot

Successful Response (200 OK)

{
    "grid_definition": {
        "x_coords": [0.5, 0.52, ..., 1.0],
        "y_coords": [0.0, 0.02, ..., 1.0]
    },
    "heatmap": [
        [0.01, 0.01, ..., 0.02],
        [0.01, 0.01, ..., 0.02],
        ...
    ]
}
